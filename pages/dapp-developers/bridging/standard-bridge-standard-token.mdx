# Bridging Your Standard ERC-20 Token to OP Mainnet Using the Standard Bridge

This tutorial will explain how to bridge an ERC-20 token from Ethereum to OP Mainnet.

This tutorial uses the [`OptimismMintableERC20Factory`](https://github.com/ethereum-optimism/optimism/blob/290f58c09293cbc1ef6d99191b9dd4847451d4ee/packages/contracts-bedrock/src/universal/OptimismMintableERC20Factory.sol) contract to create an L2 representation of an existing L1 ERC-20 token.
The `OptimismMintableERC20Factory` contract creates standardized [`OptimismMintableERC20`](https://github.com/ethereum-optimism/optimism/blob/290f58c09293cbc1ef6d99191b9dd4847451d4ee/packages/contracts-bedrock/src/universal/OptimismMintableERC20.sol) tokens that can be bridged using the `StandardBridge` contracts.
Tokens created by the `OptimismMintableERC20Factory` are mintable and burnable but have no other special logic and otherwise function as basic ERC-20 tokens.

If you need more advanced functionality you can create a custom L2 token contract as long as it implements the [`IOptimismMintableERC20`](https://github.com/ethereum-optimism/optimism/blob/develop/packages/contracts-bedrock/src/universal/IOptimismMintableERC20.sol) interface.
Check out the tutorial on [Bridging Your Custom ERC-20 Token to OP Mainnet Using the Standard Bridge](./standard-bridge-custom-token.mdx) for an tutorial on creating a custom L2 token contract.

<Callout type="warning"> 
The Standard Bridge **does not** support [fee on transfer tokens](https://github.com/d-xo/weird-erc20#fee-on-transfer) or [rebasing tokens](https://github.com/d-xo/weird-erc20#balance-modifications-outside-of-transfers-rebasingairdrops) as they can cause accounting issues within the bridge contract.
</Callout>
